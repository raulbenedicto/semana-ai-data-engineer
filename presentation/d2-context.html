TÉCNICAS DE PROMPTING

6 Técnicas Provadas

Cumulativas, não exclusivas — use todas juntas. Cada uma resolve um tipo diferente de falha do modelo.

Role + Goal

Defina quem o modelo é e o que deve fazer. A persona direciona tom e vocabulário.

Na prática:

Você é um analista de dados de e-commerce especializado em SQL.

Context / Schema

Injete conhecimento de domínio: tabelas SQL, models, regras de negócio. Sem schema, o modelo inventa.

Na prática:

Tabelas: orders(total, created_at, state), customers(name, city, segment)

Few-Shot

Mostre 3 exemplos de input → output. O modelo aprende o padrão sem fine-tuning.

Na prática:

Q: faturamento SP → SELECT SUM(total)...
Q: top produtos → SELECT name, COUNT(*)...

Chain-of-Thought

Force o modelo a raciocinar passo a passo. Melhora +34% em tarefas de lógica (Wei et al., 2022).

Na prática:

Primeiro identifique as tabelas, depois construa o JOIN, depois filtre.

Constraints

Defina limites e proibições. O que NÃO fazer é tão importante quanto o que fazer.

Na prática:

Não invente dados. Use apenas as tabelas fornecidas.

Structured Output

Especifique o formato exato: JSON, XML, Pydantic. Tags XML superam JSON em +11%.

Na prática:

Retorne JSON: revenue (float), state (str), month (str)

Estas 6 técnicas são cumulativas e validadas por pesquisa. Não escolha uma — use todas juntas. A composição é o multiplicador.

POR QUE ISSO IMPORTA

A Context Window é como a memória de trabalho do modelo. Tudo que ele sabe para responder está aqui — e tem limite. Cada camada compete por espaço. Mais tokens = mais ruído. O segredo é máximo sinal, mínimo ruído.

CONTEXT ROT

O Contexto Apodrece — Attention Budget

Precisão não cai de um penhasco — degrada gradualmente. Mais tokens = mais ruído no sinal. A atenção do transformer é n²: cada token atende a todos os outros.

“Every new token introduced depletes this budget by some amount.” — Anthropic. Trate o contexto como recurso precioso: máximo sinal, mínimo ruído.

Context Rot High Signal n² Attention
